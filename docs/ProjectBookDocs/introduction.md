ProjectBook is a Design Thinking platform for product teams. It keeps research context, problem statements, ideas, tasks, and feedback connected inside each project scope.

## Problem It Solves

Most teams lose context because work is split across docs, chat, boards, and tickets. ProjectBook reduces this fragmentation by making artifact links explicit and visible.

## Why Design Thinking Is Central

Design Thinking is the core operating model, not a label. The routes, remote functions, and artifact structure follow phase-driven progression.

## Who It Is For

- Product teams running discovery to validation loops.
- Designers and researchers who need traceable context.
- Engineers implementing solutions tied to upstream problem framing.

## What It Is Not

- Not a generic task tracker.
- Not an unstructured wiki.
- Not a service-layer architecture.
- Not a backend API product yet (current state uses in-memory data behind remote boundaries).
