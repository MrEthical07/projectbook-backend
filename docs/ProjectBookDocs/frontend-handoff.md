## 1. Required Frontend Error UI Contract

Always render these fields for every failed API call:
- status code
- name (error code)
- reason (server message)
- tip (when present; else generic support hint)

Recommended rendered shape:
```json
{
  "statusCode": 403,
  "name": "forbidden",
  "reason": "permission denied",
  "userMessage": "You do not have enough permission for this operation.",
  "tip": "If you believe this is a mistake, contact the owner of the project for requesting access."
}
```

## 2. Status Code Handling Playbook

| Status | Primary meaning | Frontend action now | Client next step |
|---|---|---|---|
| 400 | Invalid request or validation failure | Keep user on the same screen and preserve form input. | Highlight invalid fields and retry after correction. |
| 401 | Unauthenticated or invalid credentials/session | Clear stale auth for this request context and open sign-in flow. | Authenticate again, then retry. |
| 403 | Authenticated but not authorized/scope mismatch | Keep user logged in, block the action, and do not auto-retry. | Offer safe navigation and access-request option. |
| 404 | Resource not found | Stop loading and show not-found state for the target resource. | Offer refresh/back navigation. |
| 405 | Wrong HTTP method for route | Treat as client integration bug and block repeated retries. | Fix client method wiring for the endpoint. |
| 409 | Conflict with current server state | Keep local state and stop optimistic updates for this action. | Refresh server state and re-attempt if still valid. |
| 415 | Wrong content type (JSON required) | Do not retry automatically; this is a client request formatting issue. | Send the request with Content-Type application/json. |
| 429 | Rate limit exceeded | Disable submit controls for cooldown period. | Retry after cooldown. |
| 500 | Internal server error | Show failure state, keep user data, and capture telemetry. | Retry once; if still failing, escalate. |
| 503 | Dependency/service unavailable | Show temporary outage state and disable repeated immediate retries. | Retry with backoff. |
| 504 | Request timeout | Mark request as timed out and keep current UI state. | Offer manual retry with backoff. |
